Gitに関して

Gitとは

Gitは分散型バージョン管理システムと呼ばれるツール
バージョンを分散して管理しているため以下の特徴がある。
  • ファイルの変更履歴を残せる
  • 変更履歴の共有
  • 以前のバージョンに戻せる
以下の点から複数人での作業などにとても役に立つ

Gitの仕組み

Gitは以下の仕組みからなっている
  • リポジトリ　　　　　　　変更履歴を溜めておく場所
  • リビジョン　　　　　　　リポジトリに溜める変更履歴
  • ブランチ　　　　　　　　リビジョンのグループ
  • コミット　　　　　　　　リポジトリに変更した点などを送ること
  • インデックス　　　　　　コミットの対象を置いておく場所
  • ワーキングディレクトリ　リポジトリが存在するディレクトリ
  • ローカルリポジトリ　　　ローカル環境（PC内部など個人の場所）にあるリポジトリ
  • リモートリポジトリ　　　リモート環境（インターネット上など複数人が共有できる場所）にあるリポジトリ
  • プッシュ　　　　　　　　ローカルリポジトリの情報をリモートリポジトリに反映させること
  • プル　　　　　　　　　　リモートリポジトリの情報をローカルリポジトリに取り込むこと
GitHubへ反映させる方法（Mac）
ターミナルで
  1. cd mygit/xbp
  2. git status
  3. git add .
  4. git commit -m "メッセージ(日付がおすすめ)"
  5. git push
    6. の順で入れればできる。

htmlに関して

htmlとは

htmlはハイパーテキスト・マークアップ・ランゲージの略
webを作る時に使われる言語
様々なタグを用いて組み立てる

タグの種類

  • <div> 段落など塊を括るタグ
  • <img src="ファイルパス" width="スクリーンに対する割合"> 画像を貼るタグ
  • <ol><ul> 箇条書きにするタグ　oが番号付き　uが番号なし
  • <a href="リンクまたはフォルダパス">サイトの名前</a><br> リンクを貼るタグ　" target="_blank"を入れると新しいタブで開く 「../」はひとつ上の階層を意味する
  • <xmp> タグをweb上で表示する
  • <html> HTML文書のタグ
  • <head> タイトルなどを設定する場所のタグ
  • <body> 文書の本体のタグ
  • <title> webのタイトルを決めるタグ
  • <h1>-<h6> 見出しのタグ　1~6で文字のサイズが変わる
